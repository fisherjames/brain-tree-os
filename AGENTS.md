# Brian Agent Guide

## Session Start
- Read `brian/index.md` and `brian/execution-plan.md` before making non-trivial changes.
- Prefer small, verifiable edits over speculative rewrites.

## Working Rules
- Keep decisions in brain files, not only in chat history.
- Update the matching brain note when architecture, priorities, or risks change.

## Commit Policy
- Treat commits as feature-length changes, not tiny fragmented edits.
- Always call out breaking changes explicitly in commit messages.

## Verification
- Run the narrowest realistic check for the files you changed.
- If verification is incomplete, record the gap in the handoff.
